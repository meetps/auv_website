Software Sub-Division

Introduction

The software system is responsible for mission planing and all the higher level functionality. Matysa runs a custom middeware to provide a neat interface for the applications. The linux kernel provides a complicated interface for safe interprocess communication. It would be very tedious for every software developer in the Matsya team to manage safe interprocess communication and device IO.

Middleware

Our middleware abstracts out

  1. Interprocess Communication
  2. Device IO

Matsya's middleware is based on the philosophy of the Internet. The Matsya software system consists of 3 applications

  1. Mission Planner : This process runs the mission planning algorithms and maintains the state of the machine
  2. Image Processor : This process receives the state of the machine from the mission planner and runs the corresponding image processing algorithm
  3. Communicator : This process is the abstract interface to the other micro controller boards in the system

Debugging Platform

Matsya's debug platform provides on a web-based interface. A web-server (Apache 2) runs on the Pandaboard and listens for user requests. A user can send requests through any browser. This means that any person with a browser and a network access can send requests to Matsya debugging platform
